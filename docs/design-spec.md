# Claude Desktop 设计规范

## 设计目标
- 保持“终端工作台”气质：深色背景、低饱和描边、青绿色作为主操作色。
- 页面像设计稿里的组件库一样统一：同类卡片、按钮、输入框的圆角、边框、间距保持一致。
- 信息层级要克制：标题负责说明当前位置，状态和操作放到标题右侧，不额外堆说明文案。

## 布局规则
- 页面外层使用深色径向渐变背景，内容区最大宽度保持 `max-w-[1600px]`。
- 主内容左右留白使用 `px-6`，上下留白使用 `py-6`。
- 顶部标题卡片使用 `rounded-2xl`、`border border-[var(--border-color)]`、`bg-[var(--bg-secondary)]/92`。
- 标题卡片内采用左右结构：左侧主标题，右侧放状态胶囊和主按钮。
- 不在标题区堆长说明；说明文字优先放到空状态或具体表单附近。

## 圆角规则
- 大容器、页面头部、项目卡片、空状态：统一 `rounded-2xl`。
- 按钮、输入框、分组 tab、状态块：统一 `rounded-xl`。
- 状态标签如果是独立小徽标，可使用 `rounded-full`，但标题右侧统计块使用 `rounded-xl`，避免和按钮形状割裂。

## 颜色规则
- 主操作按钮使用 `bg-[var(--accent-cyan)]`，文字使用深色 `text-[#0d1117]`。
- 成功/启动使用 `var(--accent-green)`，危险/删除/停止使用 `var(--accent-red)`。
- 文本层级：主标题 `text-[var(--text-primary)]`，辅助信息 `text-[var(--text-secondary)]`，弱提示 `text-[var(--text-muted)]`。
- 边框统一使用 `var(--border-color)`，不要单独写新的灰色值。

## 组件规则
- 项目卡片保持 `rounded-2xl` + 深色线性渐变背景，不改成亮色卡片。
- 分组区保持轻量，不做成比项目卡片更重的视觉层级。
- 分组名称直接在分组 tab 内编辑，像项目卡片标题一样，不额外放一块独立表单。
- 分组删除入口放在分组 tab 右侧，用轻量 `×` 按钮；不要新增沉重的管理弹窗。
- 表单标签可以使用英文大写小字和字距，但不要放在主标题前面当装饰性 kicker。
- 主按钮文案保持短句，例如“新增项目”“启动”“关闭”。
- 下拉选择器不要裸用系统默认外观；必须使用 `appearance-none`、暗色背景、自定义箭头，并与按钮保持 `rounded-xl`。

## 交互规则
- 页面切换时，正在运行的终端不应该被销毁；能隐藏就不要卸载。
- 运行态、分组、项目配置要能被用户直观看到，避免藏在二级弹窗里。
- 新增功能前先判断它属于页面级操作、分组级操作还是卡片级操作，再放到对应层级。

## 后续开发要求
- 任何影响可见 UI 的改动，先阅读本文件。
- 优先复用现有 CSS 变量和 Tailwind 结构，不新增一套视觉语言。
- 如果确实要打破规范，必须在变更说明里解释原因。
